Cheatsheet for Git Commands

Command:git init

Der Befehl erzeugt ein Unterverzeichnis .git, in dem alle relevanten Git-Repository-Daten enthalten sind, also ein Git-Repository Grundgerüst. Zu diesem Zeitpunkt werden noch keine Dateien in Git versioniert. In Kapitel 10, Git Interna, finden Sie weitere Informationen, welche Dateien im .git Verzeichnis enthalten sind und was ihre Aufgabe ist.

Command:git add

Der Befehl gefolgt von einem Punkt(git add .) wählt alle Dateien des Projektes aus, um sie dem Index hinzuzufügen.

Command:git commit

Die Änderungen werden an den Index-Bereich übergeben. Man spricht auch vom Stage / staging. Mit dem Parameter '-m' wird eine Nachricht hinzugefügt, die die Änderungen beschreiben sollen.

Command:git push

Die Änderungen werden an Remote-Repository übergeben.

Command:git log

Der Änderungsverlauf wird je nach verwendetem Parameter angezeigt.

Command:git status

Gibt über den aktuellen Verarbeitungsstand Auskunft. Je nachdem ob Änderungen vorliegen, werden Informationen hierzu ausgegeben.

Command:git branch

Wird verwendet um einen neue Branch zu erstellen.

Command:git checkout

Wird verwendet um in einen Branch zu wechseln.

Command:git pull

Wird verwendet um Änderungen aus dem Online-Repository abzuholen.

Command:git merge

Die Änderungen eines Branch werden auf einen anderen Branch übertragen.
